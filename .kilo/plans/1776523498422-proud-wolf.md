# Lua Code Analysis & Keyframe Visualization System

## Executive Summary

Build a comprehensive system to analyze Fusion Lua code, identify custom functions, and provide an interactive keyframe/spline visualization with timeline scrubbing. This will work for both current implementation and future React migration.

## Current Infrastructure Analysis

### ✅ What Already Exists (nodes-system.js)

1. **Lua Parser** (`parseFusion()`)
   - Parses Fusion Lua format
   - Extracts BezierSplines with keyframes
   - Identifies node connections

2. **Keyframe Extraction** (`parseBezierKeyframes()`)
   - Parses frame, value, RH/LH handles
   - Supports hold/interpolation types

3. **Spline Rendering** (`renderMiniSpline()`, `renderSplineGraph()` )
   - Canvas-based bezier curve visualization
   - Shows keyframe markers
   - Grid and axis labels

4. **Parameter Panel** (`renderParams()`)
   - Displays parameters with animation indicators
   - Shows spline preview for selected parameter
   - Lists keyframe count and frame range

5. **Node Graph System**
   - Node selection → parameter selection flow
   - Side panel integration
   - Graph viewport with pan/zoom

### ❌ What's Missing

1. **Spline Evaluation Function** - interpolate value at any frame
2. **Timeline/Scrubber Component** - UI for frame navigation
3. **Custom Function Detection** - identify non-default functions
4. **Real-time Value Display** - show parameter values at current frame
5. **Playhead on Splines** - visual indicator of current frame

## React Migration Path

The architecture is designed to be framework-agnostic:

### Current (Vanilla JS)
```javascript
// Direct DOM manipulation
const scrubber = new TimelineScrubber(container, options);
```

### Future (React)
```jsx
// Wrapper components
<TimelineScrubber 
  keyframes={keyframes}
  onFrameChange={handleFrameChange}
  currentFrame={currentFrame}
/>

<SplineGraph 
  splines={splines}
  currentFrame={currentFrame}
  playheadColor="#6c7bff"
/>

<ParameterValues 
  node={node}
  currentFrame={currentFrame}
  evaluate={evaluateNodeAtFrame}
/>
```

**Migration Strategy:**
1. Core logic (spline evaluation, Lua analysis) stays pure JS
2. UI components get React wrappers
3. State management moves to React hooks/Context
4. Canvas rendering stays imperative (performance)
